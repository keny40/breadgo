# BreadGo Phase 36 지도 기반 상품 탐색 UI

## 작업 목적

Phase 36은 고객 상품 탐색 화면(`/products`)에 지도 기반 UX를 추가하는 단계입니다.

이번 단계에서는 실제 Kakao Map, Naver Map, Google Maps 같은 외부 지도 API를 연동하지 않고, API 키 없이 동작하는 MVP용 지도 패널을 구현했습니다.

## 변경 내용

- `/products` 화면에 `지도 보기` 패널 추가
- 기존 상품 리스트 유지
- 데스크톱에서는 지도 패널과 상품 리스트가 2단으로 배치
- 모바일에서는 지도와 리스트가 세로로 쌓이도록 처리
- 브라우저 현재 위치를 가져온 경우 `내 위치` 표시
- 좌표가 등록된 매장을 지도 마커 형태로 표시
- 좌표가 없는 매장은 지도에서 제외하되 상품 리스트에는 계속 표시
- 지도 마커 또는 매장 요약 클릭 시 해당 매장 상품 카드 강조
- 기존 `내 위치로 찾기` 버튼 유지
- 기존 지역 검색, 예약 생성, 수령 방식 선택, Mock 결제 흐름 유지

## 구현 방식

- 백엔드 DB/API 구조는 변경하지 않았습니다.
- 기존 `/api/v1/regions/products` 응답으로 상품 목록을 유지합니다.
- 기존 `/api/v1/regions/stores` 응답에서 매장 위도/경도를 가져와 MVP 지도 패널에 사용합니다.
- 지도는 실제 지도 타일이 아닌 CSS 기반 mock map 패널입니다.
- 외부 지도 API 키가 필요하지 않습니다.

## 검증 결과

```powershell
cd frontend
npm run lint
npm run build
```

두 명령 모두 통과해야 합니다.

## 수동 확인 항목

1. `/products` 접속
2. 지역 상품 검색
3. 지도 패널이 표시되는지 확인
4. 좌표가 있는 매장이 지도 마커로 표시되는지 확인
5. 마커 또는 매장 요약 클릭 시 상품 리스트의 해당 매장 카드가 강조되는지 확인
6. `내 위치로 찾기` 클릭 시 내 위치 표시가 나오는지 확인
7. 모바일 폭에서 지도와 리스트가 세로로 쌓이는지 확인
8. 상품 예약/수령 방식/Mock 결제 흐름이 기존처럼 동작하는지 확인

## 현재 한계

- 실제 지도 타일이 아닙니다.
- 실제 도로/건물/상권 정보는 표시하지 않습니다.
- nearby 상품 API 응답에는 매장 좌표가 직접 포함되지 않으므로, 지도 마커는 현재 선택 지역의 매장 좌표를 기반으로 표시합니다.
- 좌표가 등록되지 않은 매장은 지도에 표시되지 않습니다.
- 거리 계산과 지도 위치 시각화는 MVP 수준의 단순 표현입니다.
